# Fork the Template

Your project needs its open home first: a repo of your own on GitHub.

## What forking means

**Forking** copies someone else's repo into your account. You get all the folders and files; your copy is yours to change; the original is untouched. It is how open projects reproduce — nobody asks permission, and nothing can break.

## Get a GitHub account

If you do not have one: [github.com/signup](https://github.com/signup). Free, and the phone browser works fine for all of this.

## Fork a project repo

1. Open a PlayfulProcess project repo in your browser — this one (nara), or [kali-paradevi](https://github.com/PlayfulProcess/kali-paradevi), or any sibling project that is public. (If the one you want is still private, you will not see a Fork button — pick a public sibling, or just create a new repo at [github.com/new](https://github.com/new) and add the folders below yourself.)
2. Tap **Fork** (on a phone it may hide behind the "…" menu at the top of the repo page).
3. GitHub asks for a name. Name it after YOUR project — the subject you actually want to spend time with (`iching-altar`, `saints-of-my-grandmother`, `mushroom-field-guide`…).
4. Tap **Create fork**. Done — you have a repo.

## What the folders mean

The template's structure is the working method, so keep it:

- `grammars/` — the data of your symbolic system (this fills up later, mostly written for you).
- `research/` — notes, sources, quotes you gather. Honest sourcing lives here.
- `journal/` — one file per working day. The project's memory (chapter 5 is about why this matters most).
- `course/` — if you ever want to teach your path, chapters go here.
- `docs/` — design notes, decisions.

## What to rename

Only the words, not the structure:

- The repo name (done above — or later via **Settings → Rename**).
- `README.md`: change the title and the first paragraph to say what YOUR project is. You can edit it right in the browser — open the file, tap the pencil icon, type, then **Commit changes**. Delete the sections that are about the old project; keep the humility-charter spirit if it speaks to you.

Everything else can wait. A fork with a renamed README is a real project.

---

**Do it now:** fork (or create) your repo, then edit `README.md` so the first line names your project in your own words. One sentence is enough.
